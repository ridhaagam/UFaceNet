## Venue Fit

ACCV topics include face and gesture, biometrics, generative models for computer vision, 3D computer vision, datasets and performance analysis, recognition, and shape representation. UFaceNet fits if the work is framed as a unified face understanding and reconstruction model with strong evaluation, not just a demo.

## Candidate Title

UFaceNet: Unified Facial Analysis and Reconstruction with Task-Conditioned Face Tokens

Alternative:

UFaceNet: Extending Unified Facial Analysis to Face Reconstruction and Generation

## Core Contributions

Contribution 1:

A unified architecture that extends FaceXFormer-style task-token facial analysis with a first-class face reconstruction/generation task family.

Contribution 2:

A reconstruction-consistency block that ties generated/reconstructed outputs to identity, geometry, parsing, landmarks, pose, expression, and semantic attributes.

Contribution 3:

A benchmark packet for unified face analysis plus reconstruction, reporting original FaceXFormer task metrics together with rFID, FID-face, identity preservation, geometry error, task consistency, runtime, and ablations.

Contribution 4:

A practical analysis of the tradeoff between unified analysis performance, reconstruction quality, and real-time efficiency.

## Paper Skeleton

1. Introduction
   - Unified face analysis has matured, but reconstruction/generation remains outside the shared tokenized block.
   - FaceXFormer shows task-token unification for ten facial analysis tasks.
   - UFaceNet asks whether reconstruction/generation can become a first-class task token without degrading the original tasks.

2. Related Work
   - Single-task face analysis.
   - Multi-task face analysis and FaceXFormer.
   - 3D face reconstruction: DECA, MICA, EMOCA, 3DDFA-V2, FaceVerse-style models.
   - Face generation and identity-preserving generation.
   - Evaluation metrics: FID, rFID, identity consistency, geometry benchmarks.

3. Method
   - FaceXFormer recap.
   - UFaceNet task registry.
   - FRec token and optional geometry/texture/render tokens.
   - Reconstruction-consistency block.
   - Reconstruction/generation decoder choices.
   - Losses and multi-task schedule.

4. Experiments
   - Datasets and licenses.
   - Original task metrics.
   - Reconstruction and generation metrics.
   - Runtime and model size.
   - Ablations.
   - Robustness and bias.

5. Discussion
   - What reconstruction improves.
   - What reconstruction hurts.
   - Limits of rFID and FID-face.
   - Responsible use.

6. Conclusion

## Risk Register

Risk: local FaceXFormer code lacks full training release.

Mitigation: implement UFaceNet as a reproducible extension with explicit baseline reproduction limits and use paper metrics as external reference.

Risk: reconstruction improves rFID but damages identity.

Mitigation: identity cosine and ArcFID are required keep/discard gates.

Risk: face generation branch becomes too slow.

Mitigation: report lightweight reconstruction branch separately from optional generative refiner.

Risk: no 3D ground truth dataset is available locally.

Mitigation: use paired 2D metrics first, add NoW/MICC when data access is complete, and document blockers precisely.

Risk: broad task scope dilutes novelty.

Mitigation: center paper on FRec token plus reconstruction-consistency block, not on reimplementing every FaceXFormer task.
